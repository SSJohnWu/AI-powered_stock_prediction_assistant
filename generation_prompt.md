# 📝 AI 生成提示詞 (Prompt) 指南

如果您希望讓 AI 助手（例如 Gemini, ChatGPT 或其他 AI 程式助手）從頭到尾自動寫出與本專案功能與外觀完全相同的程式，您可以使用以下精心設計的結構化 Prompt。

---

## 📌 如何使用此 Prompt？
1. 在您的專案目錄中建立一個空的 `app.py` 檔案。
2. 複製下方框線內的 Prompt，並貼給您的 AI 助手。
3. AI 將會生成對應的 `app.py` 與 `requirements.txt`。

---

## 🤖 複製下方的 Prompt 送給 AI：

```text
請幫我用 Python 與 Streamlit 寫一個名為「AI 個股預測小幫手」的網頁應用程式，並同時提供對應的 `requirements.txt` 檔案內容。
要求使用以下技術棧與具體功能設計：

### 🛠️ 技術棧要求：
1. UI 框架：Streamlit (排版需美觀、寬螢幕佈局 `layout="wide"`)
2. 股市數據：yfinance (取得歷史股價)
3. 數據與計算：pandas, numpy, scikit-learn
4. 互動式圖表：plotly.graph_objects
5. 深度學習框架：PyTorch (用於 LSTM 模型實作)
6. 大語言模型 SDK：使用最新官方建議的 `google-genai` (注意：請勿使用已過期的 google-generativeai 包)

### 🖥️ 功能與畫面配置細節：

1. 🧭 【側邊欄 (Sidebar)】：
   - 🎯 「選擇要分析的股票」區塊：
     - 提供四個快速點擊按鈕：台積電 (2330.TW)、鴻海 (2317.TW)、AAPL、NVDA。
     - 提供一個文字輸入框 `股票代碼`，預設為 "2330.TW"。並在下方加註提醒提示。
   - 🔑 「Gemini API Key」密碼輸入框。
   - 底部有兩個按鈕：「🚀 開始預測」與「⏹️ 清除快取」。點擊開始預測時才觸發主畫面渲染。

2. 📊 【區塊 1：抓取歷史資料】：
   - 使用 `yfinance` 抓取所選個股近 1 年的歷史資料，並使用 Streamlit 的 `@st.cache_data` 做快取。
   - 資料取得成功後顯示包含以下資訊的指標 (metrics)：
     - 公司名稱、產業類型、最新收盤價、52週最高價、52週最低價。
   - 若資料少於 30 筆，顯示錯誤並中止執行。

3. 🤖 【區塊 2：AI 模型預測】：
   - 提供兩個 AI 模型進行預測與對比，並實作自迴歸滾動預測（autoregressive recursive forecast）以預測未來 20 天的走勢：
     1. 「線性迴歸 (Linear Regression)」：使用 scikit-learn 的 `LinearRegression` 進行模型訓練與預測。
     2. 「LSTM (長短期記憶網路)」：使用 PyTorch 建立一個簡單的 LSTM 模型（例如包含一個 LSTM 層與一個全連接層），進行資料正規化（如使用 MinMaxScaler 手寫或簡單除以最大值將資料縮放到 0-1 區間），訓練若干 epochs，並在預測後還原尺度。
   - 預測特徵（Window）：均用過去 30 天的股價預測當天股價。
   - 資料分割與未來預測：前 80% 作為訓練集，後 20% 作為測試集。另外，以歷史資料最後 30 天作為初始 Window，向後滾動預測未來 20 天（不含歷史資料內）的預測值。
   - 使用 Plotly 繪製折線圖：藍色實線表示「實際收盤價」，紅色虛線表示測試集區間的「線性迴歸預測值」，綠色虛線表示測試集區間的「LSTM 預測值」，橘色實線（含標記）表示「線性迴歸未來 20 天預測」，紫色實線（含標記）表示「LSTM 未來 20 天預測」。

4. 📈 【區塊 3：模型表現評估與未來走勢】：
   - 分別計算並顯示「線性迴歸」與「LSTM」兩者在測試集上的 MAE（平均絕對誤差）、RMSE（均方根誤差）以及預測漲跌方向的「方向準確率」（以 Columns 排版並排呈現，方便對比）。
   - 用一個表格顯示測試集最後 20 天的資料對比，欄位包含：日期、實際、線性迴歸預測、線性迴歸誤差%、LSTM 預測、LSTM 誤差%。
   - 用另一個表格顯示未來 20 天的預測值細節，欄位包含：日期、線性迴歸預測值、LSTM 預測值。

5. 🤖 【區塊 4：Gemini 金融分析師報告】：
   - 若使用者沒有填寫 Gemini API Key，顯示警告提示填寫。
   - 若已填寫，初始化 `genai.Client(api_key=...)`，使用 `gemini-2.5-flash` 模型。
   - 將測試集表現指標、最後 20 天實際與預測對比、以及未來 20 天的預測走勢表格製作成 Prompt，請 AI 扮演「資深全球股市分析師」，在 200 字內分析與比較兩個模型在測試集上的表現、未來 20 天走勢以及提供具體的買賣建議。
   - 加入 `try-except` 捕獲 API 呼叫異常，在網頁上顯示錯誤，並使用 `traceback.print_exc()` 將詳細的錯誤堆疊列印到後台終端機。

6. 💡 【區塊 5：學習筆記】：
   - 使用 `st.expander` 製作一個展開摺疊區，簡介線性迴歸與 LSTM 的模型原理以及它們的區別。

### 📄 輸出檔案格式：
1. **`app.py`**：請提供完整的 Python 原始碼，包含完整的 PyTorch LSTM 模型定義與訓練循環，不要使用 placeholder 或省略任何程式碼段落。
2. **`requirements.txt`**：請列出所有運行 `app.py` 所需的第三方套件名稱（包含 `google-genai`、`torch`、`scikit-learn`，不要使用已過期的 `google-generativeai`）。
```

---

## 💡 為什麼這個 Prompt 能產生好的代碼？
* **技術棧指明**：指名使用 `google-genai`，可以防止 AI 使用已被淘汰的 `google.generativeai` 舊 SDK。
* **機器學習與深度學習結合**：要求使用 scikit-learn 的線性迴歸模型與 PyTorch 實作輕量 LSTM 模型，展現傳統統計預測與現代深度學習的對比。
* **明確的畫面架構**：將整個專案切割成 5 個區塊（側邊欄、資料抓取、模型、評估、Gemini 分析、筆記），AI 可以精準地分模組撰寫，不會遺漏功能。
* **出錯診斷設計**：要求加入 `traceback` 機制，確保在使用者的開發環境出錯時容易進行排查。
